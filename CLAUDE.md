# Абитура — заметки для разработки

Offline-тренажёр (Duolingo-style) подготовки к вступительным. Чистые HTML/CSS/JS,
**без сборки и зависимостей** — файлы подключаются `<script>`-тегами и работают с file:// и GitHub Pages.

## Архитектура

Порядок загрузки скриптов в `index.html` важен (общая глобальная область):

| Файл | Содержимое |
|---|---|
| `data_math.js` `data_inf.js` `data_rus.js` | курсы: `*_COURSE` (units→theory/pool/gens) + `*_GENS`; здесь же общие хелперы `rnd/pick/fmtNum/frac` (в data_math) |
| `glossary.js` | `GLOSS`/`GLOSS_RE` — справочник терминов, автоподсветка в теории |
| `icons.js` | `ICON` + `ic(name)` — единственный источник иконок (SVG, currentColor). Эмодзи в UI-хроме не используем |
| `data_morse.js` | экспериментальный курс (после icons.js: его теория зовёт `ic()` при загрузке) |
| `core.js` | реестр курсов `COURSE_DEFS`→`COURSES`/`GENS`/`SUBJ_MONO`, прогресс `P` (localStorage), уровни `LEVEL_TIERS`, SRS-повторения, `isCorrect`, единый рендер вопроса `questionBodyHtml/collectAnswer`, тема/палитра/фон, `topbar`, справочник |
| `screens.js` | главная, карта курса, теория, квиз (`S`), умная тренировка, блиц (`B`), «Что нового» |
| `exam.js` | пробный экзамен (`EX`) |
| `stats.js` | статистика, настройки, **старт приложения** (последние строки) |
| `ai.js` | ИИ-генератор заданий (Claude API), за тумблером `showAI` |

## Инварианты — не ломать

- **Ключ localStorage `altgtu-trainer`** — легаси-имя, менять нельзя (потеряется прогресс пользователей).
- Формат задания: `{ text, type:'mc'|'input'|'multi'|'match', options?, correct, expl, hard?, set?, check?, morse? }`.
  mc: correct — индекс; input: строка или массив допустимых; multi: массив индексов; match: `pairs:[[l,r]]`, правые части уникальны.
- В экзаменах (`exam:[]`) — только mc/input.
- `MAX_LEVEL=4`, `LEVEL_TIERS` — Разминка/Практика/Сложные/Вступительный с порогами точности 50/60/70/80.
- Ошибки (`P.mistakes`) живут по Лейтнеру: `box` 1..5, `due`, интервалы `SRS_INT=[1,2,4,7,14]`.
- Русские термины в теории подсвечивает `linkTerms(el, cat)` — работает по текстовым узлам, HTML не ломает.

## Как добавить

- **Задание**: объект в `pool` нужной темы. Пометь `hard:true` для уровней «Сложные/Вступительный».
- **Генератор**: функция в `*_GENS`, вернуть объект-задание; имя вписать в `gens` темы. Прогоняется тестами ×300.
- **Тему**: объект в `units` курса (id, title, icon, theory[], pool[], gens[]).
- **Курс**: новый `data_xxx.js` (`XXX_COURSE` c `mono:'..'` + `XXX_GENS`) → script-тег в `index.html` → одна строка в `COURSE_DEFS` (core.js). Для экспериментального укажи `experimental:'showXxx'` и добавь тумблер в настройках (stats.js).
- **Иконку**: path в `ICON` (viewBox 24, stroke=currentColor).
- **Палитру/фон**: блок переменных в style.css (`:root[data-palette=..]` / `[data-bg=..]`) + пункт в настройках.
- **Запись в «Что новое»**: CHANGELOG + при заметном релизе `NEWS_V++` (screens.js) — на кнопке появится «!».

## Тесты — гонять перед каждым коммитом

```bash
node tests/engine.js     # данные всех курсов + все генераторы ×300 + юнит-кейсы движка
python tests/ui.py       # смоук в реальном Chromium (нужен playwright)
```

Особенность харнесса: `const/let` не покидают `eval`, поэтому кейсы (`tests/*-cases.js`) конкатенируются в тот же eval, что и приложение.
